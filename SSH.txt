SSH(Secure Shell Protocol)
네트워크의 프로토콜 중 하나로 컴퓨터 by 컴퓨터 같은 Public Network를 통해 서로 통신을 할 때,
보안적으로 안전하게 통신을 하기 위해 사용하는 프로토콜입니다.

대표적인 사용 예
1. 데이터 전송
2. 원격 제어

FTP나 Telnet과 같은 다른 컴퓨터와 통신을 위해 사용되는 프로토콜도 있는데
SSH를 사용하는 이유는 "보안"입니다. 정보를 직접 네트워크를 통해 넘기기 때문에 누구나 해당 정보를 열어볼 수 있어 보안에
상당히 취약합니다.
반면 SSH는 보안적으로 훨씬 안전한 채널을 구성한 뒤 정보를 교환하기 때문에 보안적인 면에서 훨씬 뛰어납니다.


Private Key And Public Key
SSH는 다른 컴퓨터와 접속하기 위해 우리가 일반적으로 사용하는 비밀번호의 입력 사용X
기본적으로 SSH는 한 쌍의 Key를 통해 접속하려는 컴퓨터와 인증과정을 거칩니다.
한쌍의 Key는 Private Key, Public Key

공개키(Public Key)를 통해 메시지를 전송하기 전 암호화
공개키는 암호화는 가능하지만 복호화 불가능

비밀키(Private Key)는 절대로 외부에 노출되서는 안되는 Key, 본인의 컴퓨터 내부에 저장.
비밀키를 통해 암호화된 메시지를 복호화
